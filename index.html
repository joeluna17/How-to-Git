This is how we use GIT

In Terminal navigate to the file path where the FORKED/CLONED file resides. Example: /Desktop/GIT_HUB/ GIT/FOLDER_OF_PORJECT

Terminal: While inside of the "FOLDER_OF_PROJECT". Type: git init this will initialize a local git repositoryin the folder

Terminal: now that chages are made in the document/documents, we need to add these to the git staging area so that we can get the changed files ready for commitment to the master or alternate branch. Type: git add

Termial: Now that the files are staged we need to commit these to our desired branch so to confirm what branch we are in Type: git status to see what branch we are in. Tp change to another branch Type: git checkout someBranchName . Alternatively, you can add a branch at the same time you check out by Typing: git checkout -b newWorkingBranch

Terminal: now that we are staged and in the correct commiting branch we need commit the files as well as add a message and the reccommended process for Lambda is as follows Type: git commit -m "message containing changes."

Terminal: now that we have commited the changes, we should now push these changes from our local git to the git hub repository that we own. To do this simply Type: git push
